# HortiMulti: A Multi-Sensor Dataset for Localisation and Mapping in Horticultural Polytunnels

## A multimodal, cross-season dataset to support the development of localisation and mapping systems for horticultural robots

### Authors
Shuoyuan Xu, Zhipeng Zhong, Tiago Barros, Matthew Coombes, Cristiano Premebida, Hao Wu, and Cunjia Liu

## Abstract
HortiMulti is a multimodal, cross-season dataset collected in commercial strawberry and raspberry polytunnels, capturing appearance variation, dynamic foliage, perceptual aliasing, and GNSS-unreliable conditions that degrade existing localisation algorithms. The sensor suite includes two 3D LiDARs, four RGB cameras, an IMU, GNSS, and wheel odometry, with ground truth derived from Total Station surveying, AprilTag fiducial markers, and LiDAR-inertial odometry. We release time-synchronised raw measurements, calibration files, survey constrainted Apriltag map, and ground truth trajectories. Some baseline benchmarks for visual, LiDAR, and multi-sensor SLAM indicate that current state-of-the-art methods remain inadequate for reliable polytunnel deployment.

## Ground Truth

Ground truth trajectories and AprilTag map [[csv](https://github.com/shuoyuanxu/HortiMulti/blob/main/TagLocations.csv)] are generated by **Poly-TagSLAM** [[code](https://github.com/shuoyuanxu/Poly-TagSLAM)], which fuses LiDAR-inertial odometry with AprilTag detections. Several Tag positions in strawberry tunnels are independently surveyed using a **Trimble S5 Total Station** to millimetre accuracy.

**AprilTag deployment:**
- **Strawberry**: 960 tags permanently mounted in a diamond formation across 8 tunnels; one tunnel with dense 2вЂ“3 m spacing for high-precision validation, remaining tunnels with sparse start and end coverage
- **Raspberry**: Paired tags clipped to flexible mounting points to accommodate unpredictable plant growth

### ROS Topics

| Sensor | Topic | Message Type | Rate |
|---|---|---|---|
| Ouster LiDAR | `/ouster/points` | `sensor_msgs/PointCloud2` | ~10 Hz |
| Velodyne LiDAR | `/velodyne_points` | `sensor_msgs/PointCloud2` | ~10 Hz |
| Forward-Left Camera | `/forwardLeft/image_raw/compressed` | `sensor_msgs/CompressedImage` | ~10 Hz |
| Forward-Right Camera | `/forwardRight/image_raw/compressed` | `sensor_msgs/CompressedImage` | ~10 Hz |
| Left Camera | `/Left/image_raw/compressed` | `sensor_msgs/CompressedImage` | ~10 Hz |
| Right Camera | `/Right/image_raw/compressed` | `sensor_msgs/CompressedImage` | ~10 Hz |
| IMU | `/ms/imu/data` | `sensor_msgs/Imu` | ~200 Hz |
| GNSS | `/antobot_gps` | `sensor_msgs/NavSatFix` | ~8 Hz |
| Wheel Odometry | `/antobot_robot/odom` | `nav_msgs/Odometry` | ~25 Hz |
| AprilTag Detections | `/<camera>/tag_detections` | `apriltag_ros/AprilTagDetectionArray` | ~10 Hz |
| TF / TF Static | `/tf`, `/tf_static` | `tf2_msgs/TFMessage` | вЂ” |

Extrinsic calibrations are embedded in the `/tf` tree for direct use in ROS-based SLAM frameworks.
