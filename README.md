# HortiMulti: A Multi-Sensor Dataset for Localisation and Mapping in Horticultural Polytunnels

## A multimodal, cross-season dataset to support the development of localisation and mapping systems for horticultural robots

### Authors
Shuoyuan Xu, Zhipeng Zhong, Tiago Barros, Matthew Coombes, Cristiano Premebida, Hao Wu, and Cunjia Liu

## Abstract
HortiMulti is a multimodal, cross-season dataset collected in commercial strawberry and raspberry polytunnels, capturing appearance variation, dynamic foliage, perceptual aliasing, and GNSS-unreliable conditions that degrade existing localisation algorithms. The sensor suite includes two 3D LiDARs, four RGB cameras, an IMU, GNSS, and wheel odometry, with ground truth derived from Total Station surveying, AprilTag fiducial markers, and LiDAR-inertial odometry. We release time-synchronised raw measurements, calibration files, survey constrainted Apriltag map, and ground truth trajectories. Some baseline benchmarks for visual, LiDAR, and multi-sensor SLAM indicate that current state-of-the-art methods remain inadequate for reliable polytunnel deployment.

The camera configuration comprises a **forward-facing stereo pair** (140 mm baseline) plus **two side-facing cameras** at 45°. Cameras and LiDARs are hardware-triggered at 10 Hz.

## Ground Truth

Ground truth trajectories are generated by **Poly-TagSLAM** [[paper](https://arxiv.org/abs/XXXX)], which fuses LiDAR-inertial odometry with AprilTag detections. Tag positions in strawberry tunnels are independently surveyed using a **Trimble S5 Total Station** to millimetre accuracy.

| Configuration | RMSE (m) | Mean Error (m) | Max Error (m) |
|---|---|---|---|
| Poly-TagSLAM only | 0.141 | 0.127 | 0.294 |
| Poly-TagSLAM + Total Station | 0.073 | 0.035 | 0.263 |
| LOO cross-validation (held-out) | **0.066** | **0.059** | 0.140 |

Leave-one-out cross-validation over 35 surveyed landmarks yields a **median held-out error of 5.0 cm**, confirming no systematic under-constraint. For multi-tunnel place recognition sequences, ground truth is derived from LIO-SAM, whose single-tunnel accuracy is validated against the AprilTag reference.

**AprilTag deployment:**
- **Strawberry**: 960 tags permanently mounted in a diamond formation across 8 tunnels; one tunnel with dense 2–3 m spacing for high-precision validation, remaining tunnels with sparse boundary coverage
- **Raspberry**: Paired tags clipped to flexible mounting points to accommodate unpredictable plant growth

## Dataset Structure

```
/Dataset
├── Feb2026/
│   ├── Strawberry_1/
│   │   ├── Strawberry_1.bag       # All synchronised sensor streams
│   │   ├── GT_trajectory.csv      # Timestamped poses (x, y, z, qw, qx, qy, qz)
│   │   └── GT_tag.csv             # Ground truth AprilTag positions
│   ├── Raspberry_1/
│   │   ├── Raspberry_1.bag
│   │   ├── GT_trajectory.csv
│   │   └── GT_tag.csv
│   ├── calibration.yaml           # Session-specific calibration
│   └── ...
├── Sep2025/
│   └── ...
└── ...
```

> ⚠️ Calibration files are provided **per recording session**, as sensor configuration may vary between deployments. Always apply the calibration file from the matching session folder.

### ROS Topics

| Sensor | Topic | Message Type | Rate |
|---|---|---|---|
| Ouster LiDAR | `/ouster/points` | `sensor_msgs/PointCloud2` | ~10 Hz |
| Velodyne LiDAR | `/velodyne_points` | `sensor_msgs/PointCloud2` | ~10 Hz |
| Forward-Left Camera | `/forwardLeft/image_raw/compressed` | `sensor_msgs/CompressedImage` | ~10 Hz |
| Forward-Right Camera | `/forwardRight/image_raw/compressed` | `sensor_msgs/CompressedImage` | ~10 Hz |
| Left Camera | `/Left/image_raw/compressed` | `sensor_msgs/CompressedImage` | ~10 Hz |
| Right Camera | `/Right/image_raw/compressed` | `sensor_msgs/CompressedImage` | ~10 Hz |
| IMU | `/ms/imu/data` | `sensor_msgs/Imu` | ~200 Hz |
| GNSS | `/antobot_gps` | `sensor_msgs/NavSatFix` | ~8 Hz |
| Wheel Odometry | `/antobot_robot/odom` | `nav_msgs/Odometry` | ~25 Hz |
| AprilTag Detections | `/<camera>/tag_detections` | `apriltag_ros/AprilTagDetectionArray` | ~10 Hz |
| TF / TF Static | `/tf`, `/tf_static` | `tf2_msgs/TFMessage` | — |

Extrinsic calibrations are embedded in the `/tf` tree for direct use in ROS-based SLAM frameworks.

## Getting Started

### Prerequisites

- ROS Noetic (Ubuntu 20.04)
- Python 3.8+

### Playing a sequence

```bash
rosbag play Feb2026/Strawberry_1/Strawberry_1.bag
```

### Evaluating a SLAM trajectory

Ground truth trajectories are provided as `GT_trajectory.csv` with the format:

```
timestamp, x, y, z, qw, qx, qy, qz
```

Timestamps align to ROS bag header time for direct comparison. Evaluation scripts for ATE and RPE compatible with the [TUM benchmark tools](https://cvg.cit.tum.de/data/datasets/rgbd-dataset/tools) are provided in `eval/`.
